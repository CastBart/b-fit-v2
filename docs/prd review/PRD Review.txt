Data Integrity Constraints:
Cascade Delete Rules
| Parent → Child | On Delete | Reasoning |
|----------------|-----------|-----------|
| User → Exercise | SET NULL | Preserve exercises if user deleted |
| User → Workout | CASCADE | User's workouts deleted with user |
| User → TrainingSession | CASCADE | User's sessions deleted with user |
| Workout → TrainingSession | RESTRICT | Cannot delete workout with sessions | 
| Exercise → WorkoutExercise | RESTRICT | Cannot delete exercise in use |
| TrainingSession → SessionSet | CASCADE | Sets deleted with session |

	- Workout -> trainingSessions: can be linked to a workout but not specifically restrict deletion of workouts. a workout can be deleted and workout session should live in the database
	- Exercise → WorkoutExercise: similar to workout -> trainingSession. a workout exercise can be linked to exercise but if an exercise is deleted, the workout exercise should not get deleted.
